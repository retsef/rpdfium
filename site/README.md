# rpdfium documentation site

Source for <https://retsef.github.io/rpdfium>, built with
[just-the-docs](https://just-the-docs.com) (Jekyll) and deployed by
`.github/workflows/pages.yml` on every push to `main` that touches `site/`.

## Local preview

```bash
cd site
bundle install
bundle exec jekyll serve --baseurl ""
# open http://127.0.0.1:4000
```

## Structure

Topic-based navigation (modeled on hexapdf's documentation):

```
site/
├── index.md              # Home
├── getting-started.md    # Installation / Usage / Feature overview
├── basics/               # Documents, text, paths, images, annotations, rendering
├── extraction/           # Tables, forms, form-aware extraction, struct tree
├── benchmarks/           # Overview + raw-text and tables results
├── architecture.md
└── assets/pdfs/example.pdf   # sample invoice used in all code snippets
```

Each page is Markdown with just-the-docs front matter (`title`, `parent`,
`nav_order`). Section parents set `has_children: true`. Site-wide settings
(theme, search, sidebar external links) live in `_config.yml`.

`assets/pdfs/example.pdf` is generated by `../benchmark/generate_pdfs.rb` —
regenerate it there, don't edit it by hand. This directory's `Gemfile` is
independent from the gem's root `Gemfile`.
