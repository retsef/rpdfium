# rpdfium benchmark suite

Reproducible benchmarks comparing **rpdfium** (latest), **pypdfium2** (raw
Python FFI binding to PDFium — the "pure PDFium speed floor"),
**pdfplumber** (the reference pure-Python extraction pipeline) and
**hexapdf** (pure-Ruby PDF library).

Three metrics per combination:

| Metric | How |
| --- | --- |
| **Execution time** | minimum of N timed runs (default 3) after a warm-up |
| **Memory** | peak RSS of the isolated runner process, max of N runs |
| **Correctness** | fraction of ground-truth data recovered (see below) |

Results are published on the documentation site under *Benchmarks*.

## Sample PDFs — 4 tiers of increasing complexity

All PDFs in `pdfs/` are synthetic, generated by `generate_pdfs.rb`
(requires the `hexapdf` gem, generation only):

| File | Pages | Complexity |
| --- | ---: | --- |
| `01_simple.pdf` | 1 | Short text + one small ruled table |
| `02_medium.pdf` | 6 | Text + one ruled table per page |
| `03_complex.pdf` | 16 | Mixed layouts: text, ruled tables, borderless columns, prestamped form |
| `04_heavy.pdf` | 60 | Dense text + a ruled table on every page |

Regenerate with:

```bash
gem install hexapdf
ruby benchmark/generate_pdfs.rb
```

## Correctness scoring

Because the PDFs are generated, the ground truth is known exactly and is
written alongside them in `pdfs/expected.json`:

- **text task** — unique sentinel tokens (`TKN-S-0001`, ...) are embedded in
  the running text; the score is the fraction recovered in the extracted
  text. Matching is whitespace-insensitive, so libraries are not penalized
  for different word-spacing reconstruction.
- **tables task** — the unique cell values of every ruled table (SKU codes
  and row totals); the score is the fraction recovered *as table cells* with
  each library's default settings.

The correctness check runs **outside** the timed section.

## Task support

| Library | text | tables |
| --- | :-: | :-: |
| rpdfium | ✓ | ✓ (built-in pipeline) |
| pypdfium2 | ✓ | — (no table layer) |
| pdfplumber | ✓ | ✓ |
| hexapdf | ✓ (via `Content::Processor`) | ✓ (via `examples/hexapdf_table_extraction.rb`) |

The "tables" task means **detecting and reading** table cells out of an
existing PDF. pypdfium2 is a raw FFI binding with no table layer, so it runs
only the text task.

hexapdf has no *built-in* table extraction, but it exposes the two primitives
the pipeline needs — per-glyph boxes (`decode_text_with_positioning`) and
stroked path segments (`Content::Processor`). `examples/hexapdf_table_extraction.rb`
is a ~120-line reference that builds a minimal lines-based extractor on them;
the hexapdf tables row uses it. It is a proof of concept (`:lines` only, one
snap epsilon, no `:text` fallback), not a peer of rpdfium's full pdfplumber
port — it matches on clean generated grids but would drop cells on dashed
rules, partial borders, or misaligned tables.

## Running

```bash
# rpdfium needs a PDFium library (or the rpdfium-binary gem)
export PDFIUM_LIBRARY_PATH=/path/to/libpdfium.{so,dylib,dll}

# Comparison runners; missing ones are skipped automatically.
pip install pdfplumber pypdfium2
gem install hexapdf

ruby benchmark/run.rb                 # full suite, Markdown table on stdout
ruby benchmark/run.rb --runs 5        # more timed runs per combination
ruby benchmark/run.rb --only rpdfium  # filter libraries
ruby benchmark/run.rb --json          # machine-readable output
ruby benchmark/run.rb 01_simple.pdf   # only specific pdfs
```

## Methodology

- Every measurement runs in its **own subprocess**, so peak RSS is clean per
  run. All four runners read **peak** RSS the same way — `getrusage`
  (`ru_maxrss`): via FFI in the Ruby runners (rpdfium, hexapdf),
  `resource.getrusage` in the Python ones (pypdfium2, pdfplumber). Apples to
  apples, not "current RSS at exit".
- Each combination gets a **warm-up run** (discarded, neutralizes OS
  page-cache effects), then N timed runs. Reported time is the **minimum**,
  reported memory the **maximum**.
- Correctness is deterministic (same input, same ground truth), so it is
  taken from the first run.
